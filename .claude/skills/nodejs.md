# Node.js Guide

## Version

Node.js **24.18.0** (pinned in `docker/dockerfile` via `ARG NODE_VERSION=24.18.0`).
Installed via `n` (version manager) during Docker image build.

## Where Node.js Is Used

This project primarily runs on **Bun**, but Node.js is used in a few specific contexts:

| Context | Why Node.js |
|---|---|
| SvelteKit production server (`src/presentation/web/build/index.js`) | `@sveltejs/adapter-node` compiles to a Node.js-compatible bundle; started with `bun run` |
| Playwright E2E test runner | Playwright requires Node.js |
| `scripts/npm-publish.sh` | Uses `node -p "require('./package.json').name"` to read JSON fields |
| Global npm tooling bootstrap in Dockerfile | `npm i -g bun npm @antfu/ni` |

## Production Startup

The compiled SvelteKit app is a standard Node.js HTTP server generated by `@sveltejs/adapter-node`:

```bash
# Started in scripts/prod.sh
bun run /workspaces/distopia/src/presentation/web/build/index.js
```

Bun is able to run the Node.js-compatible bundle directly.

## Module System

The project uses **ESM throughout** (`"type": "module"` in all `package.json` files). Node.js 24 supports ESM natively without flags.

## `@types/node`

Type definitions for Node.js APIs are pinned in the workspace catalog:

```json
"@types/node": "catalog:"   // resolves to 24.x
```

Used in packages that access Node.js built-ins (`crypto`, `fs`, `path`, `Buffer`, etc.).

## Node.js Built-ins Used in This Project

| Built-in | Where used |
|---|---|
| `crypto.randomUUID()` | PKCE session ID generation (`app-core/OAuth2.ts`) |
| `Buffer` | JWT key storage, Prisma `Bytes` field handling |
| `fs/promises` | Bot build script (`scripts/cmdGen.ts`) |
| `path` | Bot build script |
| `process.on("uncaughtException")` | Global error handler in `hooks.server.ts` |

## Updating Node.js Version

1. Update `ARG NODE_VERSION=<new>` in `docker/dockerfile`
2. Update `"@types/node"` version in `workspaces.catalog` in root `package.json`
3. Rebuild the Docker image: `cd docker && docker compose build`
