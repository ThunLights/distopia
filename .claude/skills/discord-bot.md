# Discord Bot Guide

Package: `presentation-bot` — `src/presentation/bot/`
Library: discord.js v14

## Architecture Overview

```
handleClient(client, core)          ← entry point (called from hooks.server.ts)
  ├── InteractionCreateHandler      ← routes all interactions to the right handler class
  │   ├── ChatInputCommand/         ← slash commands
  │   ├── Button/                   ← button interactions
  │   ├── Modal/                    ← modal submit interactions
  │   ├── UserSelectMenu/
  │   ├── RoleSelectMenu/
  │   └── StringSelectMenu/
  ├── MessageCreateHandler          ← message events (invite link blocking, activity tracking)
  └── GuildMemberAddHandler         ← new member events
```

`AppCore` (`this.core`) is injected into every handler and provides access to all application services.

## Base Class Hierarchy

```
Base<T, R>
  ├── CommandInteractionBase<O, T, R>
  │   ├── ChatInputCommandBase<Options>       ← slash commands
  │   └── MessageComponentInteractionBase     ← buttons / select menus
  │       └── ButtonInteractionBase
  ├── ModalSubmitInteractionBase<Options>
  ├── RoleSelectMenuInteractionBase
  ├── StringSelectMenuInteractionBase
  └── UserSelectMenuInteractionBase
```

Every handler class has access to:
- `this.core` — `AppCore` instance (all services)
- `this.parseGuild(interaction)` → `Guild | GuildParseError`
- `this.parseUser(interaction)` → `User`
- `this.checkPermission(interaction)` → called automatically before `exec()`

## Permission Guards

Three independent permission levels, each defaults to `[]` (no requirement):

```typescript
public override requireBotGuildPermissions: PermissionResolvable[] = ["SendMessages"];
public override requireBotChannelPermissions: PermissionResolvable[] = ["EmbedLinks"];
public override requireUserGuildPermissions: PermissionResolvable[] = ["Administrator"];
```

`checkPermission()` runs before `exec()` automatically — return `PermissionError` is handled by the base class.

## Slash Command Lifecycle

1. `match(interaction)` — checks `interaction.commandName === this.register.name`
2. `checkPermission(interaction)` — validates bot and user permissions
3. `parseOptions(interaction)` — extracts and validates options (returns `ValidateResult<Options>`)
4. `exec(interaction, options)` — implements the command logic

## Registering Commands

Slash commands are registered with Discord on bot startup in `handleClient()`:

```typescript
client.on("clientReady", async (client) => {
  const commands = interactionCreateHandler.commands.chatInput.map((c) => c.register);
  await client.rest.put(`/applications/${client.user.id}/commands`, { body: commands });
});
```

The `register` property shape (`RESTPostAPIChatInputApplicationCommandsJSONBody`) defines the name, description, and options that appear in Discord.

## Auto-generated Registry Files

`*.auto.ts` files in `src/EventHandler/InteractionCreateHandler/` are generated by `bun run build` (via `scripts/cmdGen.ts`). They import every handler in a subdirectory and export them as `commands`.

**Never edit `*.auto.ts` manually.** Always run `bun run build` after adding or removing a handler.

## Activity Status

Set in `clientReady` event:

```typescript
client.user.setActivity({
  name: `${client.guilds.cache.size}server | ${client.users.cache.size}users | distopia.top`,
  type: ActivityType.Playing,
});
```

## Adding a New Event Handler

1. Create a class extending `BaseHandler<HandlerSignature>` in `EventHandler/`
2. Implement `handle(...args)`
3. Instantiate it in `handleClient()` in `src/index.ts`
4. Register the event listener: `client.on("eventName", async (...) => await handler.handle(...))`

## Intents

Intents must be configured when creating the `Client` instance (in `infra-discord`). If a handler fails silently, check that the required intent is enabled in both the code and the Discord Developer Portal.

## Rate Limiting

`LateLimit` (via `this.core.latelimit`) manages per-interaction, per-message, and per-bump cooldowns stored in memory. Check `src/application/core/src/LateLimit.ts` for available methods.
