# Discord Interaction Handler Guide

All interaction handlers live under:
`src/presentation/bot/src/EventHandler/InteractionCreateHandler/`

## Handler Types and Base Classes

| Directory | Base class | Auto-generated registry |
|---|---|---|
| `ChatInputCommand/` | `ChatInputCommandBase<Options>` | `ChatInputCommands.auto.ts` |
| `Button/` | `ButtonInteractionBase` | `Buttons.auto.ts` |
| `Modal/` | `ModalSubmitInteractionBase<Options>` | `Modals.auto.ts` |
| `UserSelectMenu/` | `UserSelectMenuInteractionBase` | `UserSelectMenus.auto.ts` |
| `RoleSelectMenu/` | `RoleSelectMenuInteractionBase` | `RoleSelectMenus.auto.ts` |
| `StringSelectMenu/` | `StringSelectMenuInteractionBase` | `StringSelectMenus.auto.ts` |

## Critical: Auto-generated Files

`*.auto.ts` files are **generated by `bun run build`** in the bot package. Never edit them manually.
After creating or deleting any handler file, always rebuild:

```bash
cd docker && docker compose exec app sudo bun run build
```

## Common Patterns

### Parsing the guild (required for most guild-scoped operations)

```typescript
const guild = await this.parseGuild(interaction);
if (guild instanceof GuildParseError) {
  return { content: guild.message, flags: [MessageFlags.Ephemeral] };
}
```

### Permission guard

```typescript
public override requireUserGuildPermissions: PermissionResolvable[] = ["Administrator"];
```

### Updating an existing message (buttons / modals triggered from a message)

```typescript
return await interaction.update({ content, components, embeds });
```

### Validating modal/command options with Zod

```typescript
import z from "zod";
import { validator, type ValidateResult } from "../../../utils/validator";

const Schema = z.object({ value: z.string().max(100) });
type Options = z.infer<typeof Schema>;

// in parseOptions:
return await validator({ value: interaction.fields.getTextInputValue("value") }, Schema);
```

### Accessing core services

`this.core` is injected by the base class and provides access to all application services (guild settings, user data, bump logic, etc.). Explore `app-core` for available methods.

## Page Builders (Pagination)

Paginated views live under `Page/`. They are stateless functions that return `InteractionReplyOptions`. Buttons that need to refresh a page call the relevant `page()` function and pass the result to `interaction.update()`.

## Component Factory Files

`Component/Button/` contains factory helpers used by page builders to construct `ActionRowBuilder` instances with the correct `customId`s. Add a companion factory here when a new button is part of a reusable page layout.

## Workflow for Adding a New Handler

1. Create the handler file in the appropriate subdirectory
2. Run `bun run build` to regenerate the `*.auto.ts` registry
3. Run `bun run typecheck` to verify types
4. Test manually in Discord or write a Vitest unit test
