# Distopia вЂ” Claude Code Guide

Distopia is a monorepo project combining a Discord bulletin board / utility bot with a web frontend.
Site: [distopia.top](https://distopia.top)

---

## Development Environment

**Required: develop inside the devcontainer.** Running directly on the host is not supported.

### First-time Setup

```bash
# 1. Run on the host before starting the devcontainer (once only)
docker volume create distopia-db-store

# 2. Run inside the devcontainer after it starts
sudo scripts/setup.sh
```

`setup.sh` installs dependencies and runs `bun run setup` (DB migration + build).

### Environment Variable Files

| File | Purpose |
|---|---|
| `.env` | Application code (bot token, DB URL, etc.) |
| `docker/.env` | Container port forwarding settings |

Copy `.env.example` to `.env` and fill in the values.

### Dev Ports

| Port | Purpose |
|---|---|
| 5173 | Vite dev server (SvelteKit) |
| 4173 | Vite preview |
| 3000 | Production build preview |
| 6006 | Storybook |

## Commands

All commands are run via `docker compose exec` from the `docker/` directory on the host.

```bash
cd docker

docker compose exec app sudo bun run build        # Build all packages
docker compose exec app sudo bun run typecheck    # Type check
docker compose exec app sudo bun run lint         # Lint
docker compose exec app sudo bun run format       # Format
docker compose exec app sudo bun run deploy-db    # Apply DB migrations
```

### Production Deploy (outside devcontainer)

```bash
environment/prod.sh
```

## Discord Bot Structure

Key layout under `src/presentation/bot/src/`:

- **EventHandler/** вЂ” Discord event handlers
  - `GuildMemberAddHandler.ts` вЂ” member join events
  - `MessageCreateHandler.ts` вЂ” message receive events
  - `InteractionCreateHandler/` вЂ” interaction handling
    - `ChatInputCommand/` вЂ” slash commands
    - `Button/` вЂ” buttons
    - `Modal/` вЂ” modals
    - `Page/` вЂ” pagination
    - `RoleSelectMenu/` / `StringSelectMenu/` / `UserSelectMenu/` вЂ” select menus

`*.auto.ts` files (`Buttons.auto.ts`, `ChatInputCommands.auto.ts`, etc.) are auto-generated by `bun run build`. Do not edit them manually.

## Database

- ORM: Prisma 7 (`infra-database` package)
- DB: PostgreSQL 17 (Docker container `distopia-db`)
- After schema changes, apply migrations with `docker compose exec app sudo bun run deploy-db` from the `docker/` directory.

## HTTP Infrastructure (`infra-http`)

`src/infrastructure/http` is a security-focused HTTP utility package that guards against SSRF and unsafe external requests.

### Key exports

| Export | Description |
|---|---|
| `SafeUrl` | Branded string type вЂ” marks a URL as validated for use in fetch calls |
| `safeUrl` | Tagged template literal that `encodeURIComponent`-encodes all interpolated values |
| `safeFetch` | Fetch wrapper with SSRF protection, body size limit, manual redirect handling, and timeout |
| `safeFetchForDiscord` | `safeFetch` variant restricted to Discord domains only (`discord.com`, `discordapp.com`, `discord.gg`) |
| `isLocalUrl` | Returns `true` if a URL resolves to a private/local address (blocks SSRF) |
| `isInviteLink` | Follows redirects via `safeFetch` and checks whether the final URL is a Discord invite link |
| `isUsedCf` | Detects Cloudflare challenge pages (HTTP 403 + "Just a moment..." title) |

### Error types

| Error | Meaning |
|---|---|
| `LocalAddressError` | Target URL resolved to a private/local IP |
| `InvalidDomainError` | Domain is not in the allowed list (Discord-only fetch) |
| `RedirectError` | Redirect chain exceeded `DEFAULT_MAX_REDIRECT` |
| `HeaderError` | Missing or invalid `location` header, or non-http(s) redirect target |
| `BodySizeError` | Response body exceeds the allowed size limit |

### Usage notes

- Always use `SafeUrl` / `safeUrl` template tag instead of raw strings when constructing URLs for fetch calls.
- `safeFetch` handles redirects manually: authorization and cookie headers are stripped on cross-origin redirects.
- This package has no build step вЂ” it is consumed directly as TypeScript source.

## Testing

```bash
cd docker

# Unit tests (per package)
docker compose exec app bun run test

# Web E2E tests (Playwright)
docker compose exec app sh -c "cd src/presentation/web && npx playwright test"

# Storybook
docker compose exec app sh -c "cd src/presentation/web && bun run storybook"
```

## Versioning & Releases

Changesets is used for versioning.

```bash
# Version bump (CI / release maintainers only)
bun run version
```

No Changesets interaction is needed during normal development.
